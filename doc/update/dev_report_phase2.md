### 2.5 模型选择决策

sklearn CART 在测试集上大幅优于 ID3（Macro F1 95.9% vs 88.6%），原因是：
- CART 二叉分裂更好利用 Q 特征的序数性质（A<B<C<D）
- sklearn 的 `min_samples_leaf` 和 `min_samples_split` 提供了有效正则化
- ID3 多叉树数据碎片化严重，泛化能力弱

**因此部署模型选择了 sklearn CART（转换为 JSON 二叉判定树格式），而非原始 ID3 多叉树。**

## 三、部署接口

### 3.1 文件结构

```
deploy/
├── classifier.py   # TCMClassifier 类（仅依赖 json, os, typing）
└── model.json      # sklearn CART 二叉判定树 (5172 bytes)
```

### 3.2 JSON 模型格式

```json
{
    "feature": "Q1",
    "threshold": 1.5,
    "majority": "C",
    "left": {
        "feature": "Q2",
        "threshold": 0.5,
        "majority": "A",
        "left": { "feature": null, "majority": "A" },
        "right": { ... }
    },
    "right": { ... }
}
```

- `feature`: 分裂特征名 (`Q1`–`Q10`, `F1`–`F3`)，叶子节点为 `null`
- `threshold`: 判定阈值。Q 特征值 0/1/2/3 与阈值比较；F 特征值 0/1 与阈值比较
- `majority`: 当前节点多数类（fallback 用）
- `left`/`right`: 满足 `value <= threshold` 走 left，否则走 right

### 3.3 TCMClassifier 接口

```python
clf = TCMClassifier("deploy/model.json")
result = clf.predict("DCACABAACC", [1, 1, 0])
# → "B" (气虚质)
```

- **输入校验**：`answers` 长度=10 且仅含 A-D；`features` 长度=3 且仅含 0/1
- **未见过值 fallback**：走 `majority` 分支（通过 sklearn 树的 `threshold` 全覆盖）
- **依赖**：仅 Python 标准库 (`json`, `os`, `typing`)

## 五、结论

1. **sklearn CART 模型测试集 Macro F1=95.88%，远超 85% 目标值**，每类 F1 均 ≥ 88.2%
2. **泛化能力良好**：训练集→测试集 F1 落差 0.024，过拟合程度低
3. **主要混淆方向**：阳虚→气虚(2)、气虚↔血瘀(2)、阳虚→血瘀(1)、湿热→特禀(1)，均为中医体质分类中的天然模糊地带
4. 部署分类器仅依赖 Python 标准库，`deploy/` 目录可独立拷贝至目标环境运行
5. 9 种体质标准模式端到端预测全部正确（9/9）
6. **望诊特征输入顺序**：[舌苔, 眼圈, 脸色]（F1/F2/F3）
