# AI凉茶机 — 中医体质分类决策树接口需求文档 (Proposal.md)

## 文档说明

本文档定义 AI 凉茶机项目**决策树接口**的功能需求、技术规格、输入输出规范及交付标准。项目的理论基础与数据生成标准详见 [proof.md](proof.md)。

---

## 一、项目概述

### 1.1 项目背景

本项目旨在开发一个中医体质分类接口，通过 10 个问诊问题和 3 个望诊特征，将用户分类至中医九种体质之一。茶疗建议等功能在项目其他模块中处理，不在本接口范围内。

**当前方案**：加权余弦相似度（Weighted Cosine Similarity），基于 proof.md 九种体质标准模式进行原型匹配。

**保留方案**：决策树模型（`deploy_tree/`），待真实临床数据积累后可重新启用。

### 1.2 项目目标

开发一个**可独立部署**的 Python 体质分类接口：

- **当前阶段**：余弦相似度匹配，零训练，纯 Python 标准库，直接使用 proof.md 标准模式
- **扩展阶段**：决策树模型（sklearn CART），需真实临床数据训练，已保留在 `deploy_tree/`

### 1.3 核心参考

| 参考资料 | 用途 |
|---------|------|
| [proof.md](proof.md) | 体质分类标准、问题设计、特征映射、茶疗推荐的理论依据和数据生成规范 |
| 周萍《基于中医体质理论与六大茶类的体质茶疗法成果研究》(2026) | 核心理论框架与茶疗方案来源 |
| ML_Lab1_Bank_Student_FrameworkV4.ipynb | 手写 ID3 决策树框架参考（sklearn + 纯 Python 两种实现） |

## 二、功能需求

### 2.1 输入规格

| 输入项 | 类型 | 格式 | 说明 |
|-------|------|------|------|
| 问诊回答 | `str` | 长度为 10 的字符串，仅含 `A` `B` `C` `D` | 10 个问题的回答，按 Q1-Q10 顺序排列 |
| 望诊特征 | `list[int]` | 长度为 3 的列表，仅含 `0` 或 `1` | [舌苔, 眼圈, 脸色]，对应 F1/F2/F3，0=正常, 1=异常 |

**输入示例**：
```python
answers = "CBBAAABACA"    # 10个问题的回答
features = [1, 0, 1]      # 舌苔异常, 眼圈正常, 脸色异常
```

**输入校验要求**：
- `answers` 必须为长度 10 的字符串，每个字符必须是 `A`/`B`/`C`/`D` 之一
- `features` 必须为长度 3 的列表，每个元素必须是 `0` 或 `1`
- 非法输入应抛出明确的异常信息

### 2.2 输出规格

| 输出项 | 类型 | 格式 | 说明 |
|-------|------|------|------|
| 体质编码 | `str` | 单个字母 `A`-`I` | 九种体质之一的编码 |

> **注意**：体质名称、茶疗建议等后续展示层内容由项目其他模块处理，本接口仅返回体质编码。

**体质编码映射表**：

| 编码 | 体质名称 | 编码 | 体质名称 | 编码 | 体质名称 |
|-----|---------|-----|---------|-----|---------|
| A | 平和质 | D | 阴虚质 | G | 血瘀质 |
| B | 气虚质 | E | 痰湿质 | H | 气郁质 |
| C | 阳虚质 | F | 湿热质 | I | 特禀质 |

采用**类封装**设计：

```python

            ValueError: 模型文件格式无效

        根据问诊回答和望诊特征预测体质编码。

            features: 长度为3的列表 (0/1)

        Returns:
            str: 体质编码 A-I

        Raises:
            ValueError: 输入格式无效
        """
        ...
```

## 三、技术需求

### 3.1 训练阶段（允许使用第三方库）

| 要求项 | 说明 |
|-------|------|
| 语言 | Python 3.x |
| 依赖库 | pandas, numpy, scikit-learn（仅训练阶段） |
| 模型类型 | 决策树分类器（CART / ID3） |
| 训练数据 | 1000 条样本，9 种体质均衡分布 |
| 数据划分 | 训练集 70% / 验证集 15% / 测试集 15% |
| 超参数调优 | 至少比较 3 组参数（criterion, max_depth, min_samples_split 等） |
| 评估指标 | Accuracy, Precision, Recall, F1-score（宏平均与按类） |
| 输出物 | 训练脚本、评估报告、序列化的 JSON 决策树文件 |

**训练流程**：
1. 数据生成（按 proof.md 标准生成 1000 条带标签样本）
2. 数据预处理（编码转换：A/B/C/D → 0/1/2/3）
3. 训练/验证/测试划分
4. sklearn 决策树训练 + 超参数调优
5. 手写 ID3 决策树训练（用于导出 JSON）
6. 模型评估与对比
7. 将较优模型导出为 JSON 文件

### 3.2 部署阶段（纯 Python + JSON，不依赖 ML 库）

| 要求项 | 说明 |
|-------|------|
| 语言 | Python 3.x（标准库） |
| 外部依赖 | 无（仅标准库 json, typing 等） |
| 模型文件 | 单个 JSON 文件，包含完整决策树结构 |
| 预测逻辑 | 基于 JSON 模型的自实现决策树遍历算法 |

**部署文件清单**：
```

```

### 3.3 JSON 模型格式

沿用 notebook 中手写 ID3 的字典结构，序列化为 JSON：

```json
{
    "feature": "Q1",
    "majority": "B",
    "children": {
        "A": {
            "feature": "Q2",
            "majority": "A",
            "children": {
                "A": "A",
                "B": "A",
                "C": "B",
                "D": "B"
            }
        },
        "B": "B",
        "C": {
            "feature": "Q3",
            "majority": "C",
            "children": {
                "A": "B",
                "B": "C",
                "C": "C",
                "D": "D"
            }
        },
        "D": "B"
    }
}
```

**字段说明**：
- `feature`：当前节点的分裂特征名（如 `"Q1"`, `"F1"` 等）
- `majority`：当前节点的多数类（用于处理未见过的特征值，fallback 预测）
- `children`：子节点字典，键为特征取值（`"A"`/`"B"`/`"C"`/`"D"` 或 `"0"`/`"1"`），值为子树或叶子标签

**特征名约定**：

| 特征名 | 含义 | 取值 |
|-------|------|------|
| Q1–Q10 | 十个问诊问题 | A / B / C / D |
| F1–F3 | 三个望诊特征 | 0 / 1 |

## 四、数据需求

### 4.1 问诊问题与评判标准文档

在数据生成之前，需将 10 个问诊问题及其选项、各体质的关键评判标准从 [proof.md](proof.md) 中独立提取，编写为 [question.md](question.md)，作为数据生成脚本和人工标注的直接参考。

**question.md 应包含**：
- 10 个问题的完整题干与 A/B/C/D 四个选项的描述
- 每个选项对应的体质倾向
- 九种体质的关键判别特征速查表（从 proof.md 第六节提取）
- 与 proof.md 的溯源链接

### 4.2 数据生成

| 要求项 | 说明 |
|-------|------|
| 生成标准 | 严格依据 proof.md 第六节「特征与体质对应关系总览」表 |
| 总样本量 | 1000 条 |
| 类别分布 | 9 种体质均衡，每类约 111 条 |
| 噪声策略 | 在标准模式基础上按概率随机扰动（模拟真实人群的症状变异） |
| 生成脚本 | `scripts/generate_data.py`，可复现（固定随机种子） |

**生成逻辑说明**：
1. 对每种体质，按第六节映射表生成"标准样本"（各问题的典型回答）
2. 以一定概率（建议 15%-25%）将非核心特征值随机扰动到相邻等级
3. 核心诊断特征（表中加粗 **D** 的单元格）扰动概率应更低（建议 5%-10%）
4. 控制每种体质的噪声水平使其保持类别可区分性
5. 字符型标签（A-I）与数值编码（0-8）同时输出

**输出文件**：数据生成脚本须产出以下两个 CSV 文件：

| 文件名 | 行数 | 内容 | 说明 |
|-------|------|------|------|
| `data/tcm_data_labeled.csv` | 1000 条 | 带标签全量数据 | 包含序号、13 个特征列、AI 分类标签、分类依据说明。用于模型训练 |
| `data/tcm_data_unlabeled.csv` | 100 条 | 无标签验证子集 | 从全量数据中取前 100 条（随机打乱后等效于随机抽样），仅含序号和 13 个特征列，供人工独立标注后验证 AI 标签一致性 |

> **验证策略**：人工对 unlabeled 的 100 条数据进行体质标注，然后与 labeled 中对应 id 的 AI 标签对比。目标：AI 标签与人工标签一致性 ≥ 90%。

**tcm_data_labeled.csv 字段定义**：

| 字段名 | 类型 | 说明 |
|-------|------|------|
| `id` | int | 样本序号（从 1 开始连续编号），用于标注误差分析时快速索引 |
| `Q1`–`Q10` | char | 问诊回答，取值 A/B/C/D |
| `F1`–`F3` | int | 望诊特征，取值 0/1 |
| `label` | char | AI 生成的体质分类标签，取值 A-I |
| `reason` | str | AI 分类依据说明。格式："核心匹配：{体质名}（{关键特征}）；次要表现：{特征列表}"。若为标准模式则标注"标准模式"；若含噪声则标注扰动详情 |

**tcm_data_unlabeled.csv 字段定义**：

| 字段名 | 类型 | 说明 |
|-------|------|------|
| `id` | int | 样本序号（与 labeled 文件一一对应） |
| `Q1`–`Q10` | char | 问诊回答，取值 A/B/C/D |
| `F1`–`F3` | int | 望诊特征，取值 0/1 |

> **注意**：两个文件的 `id` 序号必须一一对应，且同一 `id` 的特征数据完全相同，确保后续误差分析可精确匹配。

### 4.3 数据划分

| 子集 | 比例 | 样本量 | 用途 |
|-----|------|------|------|
| 训练集 | 70% | ~700 | 决策树模型训练 |
| 验证集 | 15% | ~150 | 超参数调优 |
| 测试集 | 15% | ~150 | 最终模型评估 |

划分策略：分层抽样（stratified split），确保各子集中九种体质比例一致。

### 4.4 标注误差分析脚本

编写 `scripts/compare_labels.py`，用于对比 AI 生成标签与人工标注标签之间的差异。

**输入**：
- AI 标签文件：`data/tcm_data_labeled.csv`（取前 100 条，即验证子集）
- 人工标注文件：`data/tcm_data_manual.csv`（格式：`id` + `label`，由人工在 `tcm_data_unlabeled.csv` 上标注后提供）

**一致性目标**：AI 标签与人工标签总体一致率 ≥ 90%。

**输出**：
1. **误差率统计**（打印到控制台）：
   - 总样本数、一致样本数、不一致样本数
   - 总体误差率（百分比）
   - 按体质类型分组的误差率（每类体质的标注一致率）

2. **差异明细文件** `data/label_diff.csv`，包含所有标注不符的样本：

| 字段名 | 类型 | 说明 |
|-------|------|------|
| `id` | int | 样本序号（方便快速定位修改） |
| `Q1`–`Q10` | char | 问诊回答 |
| `F1`–`F3` | int | 望诊特征 |
| `ai_label` | char | AI 分类标签 |
| `manual_label` | char | 人工标注标签 |
| `ai_reason` | str | AI 分类依据说明（从 labeled 文件关联） |

3. **混淆摘要**（打印到控制台）：AI 标签 vs 人工标签的 9×9 混淆矩阵（简化版，仅显示不一致的项）。

**使用方式**：
```bash
python scripts/compare_labels.py data/tcm_data_labeled.csv data/tcm_data_manual.csv
```

## 六、验收标准

### 6.1 模型性能

| 指标 | 最低要求 | 目标值 |
|-----|---------|-------|
| 测试集 Accuracy | ≥ 75% | ≥ 85% |
| 测试集 Macro F1 | ≥ 70% | ≥ 80% |
| 每类 Recall | ≥ 60% | ≥ 75% |

> **说明**：由于部分体质之间症状存在天然重叠（如气虚与阳虚均可出现疲劳），100% 分类准确率在理论上不可达。上述指标基于 9 分类任务的实际可达水平设定。

### 6.2 接口功能

- [ ] `TCMClassifier` 类可成功加载 JSON 模型文件
- [ ] 输入校验：非法输入抛出明确异常
- [ ] 单条预测返回体质编码（A-I）
- [ ] 面对未见过特征值时可 fallback 到多数类，不崩溃
- [ ] 部署目录 `deploy/` 可独立拷贝到目标环境运行（无需安装 sklearn/pandas）

### 6.3 数据与脚本

- [ ] 数据生成脚本可复现（固定随机种子），产出一对 `tcm_data_labeled.csv` 和 `tcm_data_unlabeled.csv`
- [ ] 两个 CSV 的 `id` 序号一一对应，特征数据完全一致
- [ ] 九种体质样本量均衡
- [ ] 标注误差分析脚本 `compare_labels.py` 可正常运行，输出误差率和差异明细
- [ ] [question.md](question.md) 内容与 proof.md 第四节一致，可独立作为标注参考
